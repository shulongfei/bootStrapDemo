本篇文章仅学习使用，刚入门时对自己造成了一些困扰的地方，也没有找到好的资料，所以做一个总结。以下为自己工作中用过，并总结出来的的常用命令。 建议新人从命令开始学更容易掌握（使用github练习）。 一、本地如何拥有一个远程项目（pc） 方式1： git clone 远端地址（记得输入用户名和密码） 方式2： 初始化一个本地仓库，主要用于新建项目时候使用。 二、add 之后的撤销操作 git reset HEAD 不接东西表示撤销所有 git reset HEAD xxx xxx 表示撤销需要撤销的文件,多个文件之间用空格隔开 commit 或 push 之后的撤销操作（新人建议练习之后操作，我的描述有可能不准确） $ git commit --amend 修改commit的注释，已最后一次修改的内容为主 $ git log 查看提交的日期和id $ git reset –-hard id 撤销commit操作，保留原有代码 $ git reset id 撤销commit操作，保留上次pull的代码（默认模式 --mixed 不保留自己修改的代码） $ git reset --soft id 适用于push代代码到远端，撤回铺设操作，保留上个版本代码（id填写前一个版本的id） git pull 发生冲突的时候应该如何解决 方式1（本人常用）： 1、先 commit 提交本地代码至缓存 git add -> git commit -m '注释内容' 2、git pull 则出现冲突文件名 文件内容则标注出冲突的内容如： <<<<<<< HEAD 本地修改 ======= 远端别人修改 >>>>>>> a94b10b314c646c482f012ea3732ddde931c549e 将文件处理之后重新执行 git add -> git commit -m '注释内容' 方式2（学习使用）： 1、保留生产服务器上所做的改动,仅仅并入新配置项 git stash -> git pull -> git stash pop 然后重新提交代码即可 git add -> git commit -m '注释内容' git stash: 将当前的工作区内容保存到Git栈中。 git stash pop: 从最近的一个stash中读取内容并恢复。 git stash list: 显示Git栈内的所有备份 方式3（不推荐使用）：直接 git reset --hard id -> git pull 恢复到远端维修改之前的版本，更新远端最新内容。
